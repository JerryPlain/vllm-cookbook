# 参数与调优清单（实战版）

下面是你未来复用时最常用的一页速查。

## 1. 启动前检查

- 模型路径是否正确（HF ID 或本地路径）。
- `TP` 是否满足 heads 整除。
- GPU 数量是否与 `CUDA_VISIBLE_DEVICES`、`TP` 一致。
- 缓存目录是否有可写权限。

## 2. 关键参数解释

- `tensor_parallel_size`
  - 控制模型横向切分，影响显存占用与通信开销。
- `max_model_len`
  - 影响可处理上下文与 KV Cache 显存。
- `gpu_memory_utilization`
  - 控制引擎可占用显存比例。
- `enable_chunked_prefill`
  - 对长上下文与吞吐稳定性有帮助。
- `enforce_eager`
  - 调试友好，环境兼容性通常更好。
- `disable_custom_all_reduce`
  - 在部分集群环境可绕过通信实现不稳定问题。

## 3. 常见报错到处理建议

- OOM
  - 降低 `max_model_len`。
  - 降低 `gpu_memory_utilization` 或减少 batch 并发。
- TP 非法
  - 先运行 `examples/04_tp_check_heads.py`。
- 首 token 很慢
  - 关注 prompt 长度和 prefill 压力。
- 吞吐不稳定
  - 固定参数后分步排查，避免多变量同时变化。

## 4. 建议的参数迭代流程

1. 基线：`TP=1` + 默认参数跑通。
2. 并行：逐步增加 `TP`，每次都做合法性检查。
3. 长上下文：调整 `max_model_len` 并观察显存。
4. 服务化：再引入 LoRA、并发与压测。

## 5. 最佳实践

- 每次改动只改一个维度。
- 每个结论都保留对应可运行脚本。
- 失败 case 也要写入文档，避免重复踩坑。
